CS Adoption Documentation

Data covers a rolling 24-month window  ·  Current month data available from the 3rd onward

DevOps

Tracks Artifactory adoption across Cloud and Self-Hosted deployments at the subscription-month level

Cloud

Data Source

Adoption metrics are sourced from the product analytics table analysts.devops_artifactory_highvalue_interactions which provides a daily True/False flag per subscription for each funnel stage (Activation, Adoption, Stickiness). Stage definitions follow the Artifactory North Star Metric documentation . Daily flags are then aggregated to the subscription-month level using the rules on the right.

Monthly Evaluation Rules
Past Months

The adoption funnel status is determined by the data recorded on the last day of that month.

Current Active Month

The most recent available data point is used, regardless of where we are in the month.

Applied Filters
  • Test accounts and subscriptions with a future start date are excluded.
  • Limited to subscriptions tied to Enterprise X or Enterprise+ opportunities.
  • For subscriptions tied to a co-termed contract, the timeline is shown up to that co-term month.

Self Hosted

Data Source

Tracked using License Utilization data from the Salesforce HUD object, pre-aggregated into a monthly history table. For each subscription-month, the highest utilization percentage across all associated HUDs is used.

Timeline: The HUD timeline includes only months where the HUD is in Active status. It runs until the subscription's expiry date if one is set, or continues as long as the HUD stays Active if no expiry is defined.

Monthly Evaluation Rules
Past Months

For each HUD, the last recorded reading of that month is used. Where a subscription has multiple HUDs, the one with the highest utilization percentage is selected.

Current Active Month

The most recent available HUD reading is used. Where multiple HUDs exist, the highest utilization is selected.

Applied Filters
  • Test accounts and subscriptions with a future start date are excluded.
  • Limited to subscriptions tied to Enterprise X or Enterprise+ opportunities (Cloud variants excluded).
  • For subscriptions tied to a co-termed contract, the timeline is shown up to that co-term month.
Adoption Funnel

Stage is determined by the license utilization percentage from the Salesforce HUD object.

Security

Tracks JAS and Curation adoption across Cloud and Self-Hosted deployments at the subscription-month level

Cloud

Data Source

Adoption metrics are sourced from the product analytics table analysts.fact_security_customers_history_adoption_metrics which provides a daily True/False flag per subscription for each JAS and Curation adoption stage. Daily flags are then aggregated to the subscription-month level using the rules on the right.

Monthly Evaluation Rules
Past Months

The adoption funnel status is determined by the data recorded on the last day of that month.

Current Active Month

The most recent available data point is used, regardless of where we are in the month.

Applied Filters
  • Only subscriptions with at least one active JAS or Curation product
Adoption Funnel

Evaluated independently per product. Stage flags are pre-computed by the product analytics team.

Grace Period: For newly added JAS or Curation products, a 60-day onboarding period applies before the subscription becomes eligible for adoption tracking.

Time to Value (TTV)

Tracks how long each product takes to reach an adoption milestone, bucketed as 0–90 days · 91–180 days · 181+ days · Pending.

Products that have not yet reached any milestone are placed in the Pending bucket.

The date axis represents the product start month, not the month the milestone was reached.

TTV 1

Days from the product start date (first day included in the subscription) to first Activation.

TTV 2

Days from first Activation to first Adoption. If Activation and Adoption are reached on the same day, TTV 2 = 0 days.

Self Hosted

Data Source

Adoption is measured using the Salesforce TaskRay object, which lives under the HUD framework. Only TaskRays with Activity type JAS or Curation are included. The Adoption Stage field in TaskRay drives all funnel logic.

Deduplication: When multiple TaskRays of the same activity exist on the same HUD in a given month, the one with the most recently updated adoption stage is used. When multiple HUDs exist on the same subscription for the same activity, the HUD whose TaskRay is at the most advanced stage is selected.

Monthly Evaluation Rules
Past Months

The adoption funnel status is determined by the TaskRay reading recorded on the last day of that month.

Current Active Month

The most recent available TaskRay reading is used.

Timeline: The HUD timeline is capped by the subscription's expiry date - months beyond that point are excluded. Within that window, the TaskRay timeline extends through the last active month of the linked HUD. If a TaskRay has recorded activity beyond that point (due to Salesforce data inconsistencies), the later activity is preserved.

Applied Filters
  • Test accounts and subscriptions with a future start date are excluded.
  • Limited to subscriptions tied to Enterprise X or Enterprise+ opportunities (Cloud variants excluded).
  • Include only active Curation or JAS products.
  • For subscriptions tied to a co-termed contract, the timeline is shown up to that co-term month.
Adoption Funnel

Evaluated independently per product (JAS and Curation). Stage is based on the last TaskRay reading of each month.

Grace Period: For new products (not present in the previous contract), the subscription becomes Eligible 60 days after the TaskRay start date. For existing products, it is eligible from the TaskRay start date immediately.

Time to Value (TTV)

TTV in Self Hosted applies to new products only.

Tracks how long each product takes to reach an adoption milestone, bucketed as 0–90 days · 91–180 days · 181+ days · Pending.

Products that have not yet reached any milestone are placed in the Pending bucket.

The date axis represents the product start month, not the month the milestone was reached.

TTV 1

Days from the TaskRay start date until the adoption stage first reaches Activation, Installation / Configuration, or Adoption Initiation.

TTV 2

Days from first Activation until the adoption stage first reaches Implementation or Stickiness.

Note: If a subscription skips Activation entirely and jumps directly to Implementation (e.g. moving from New Project straight to Implementation), TTV 1 is assigned the Implementation date and TTV 2 = 0 days.
